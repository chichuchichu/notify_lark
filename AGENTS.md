# notify_lark 通知规则

## 所有用户通知统一走飞书

在执行任务过程中，**任何时候需要通知用户、询问用户、等待用户操作**，都统一用 `notify_lark` 发送飞书消息，而不是依赖系统通知（系统通知可能被错过）。

### 触发场景

- 任务完成需要告知结果
- 需要用户做出选择或确认
- 需要用户提供额外信息
- 重要步骤完成需要汇报进度
- 遇到错误需要用户介入
- 长时间任务中途的进度汇报

### 步骤

1. 确认 notify_lark 已安装：运行 `notify_lark --help`
2. 若未安装，先执行：`cargo install --git https://github.com/chichuchichu/notify_lark.git`
3. 发送通知：
   ```
   notify_lark "<场景>: <具体内容>"
   ```
4. 消息控制在 200 字以内
5. 如果调用失败（如未配置 webhook），报错但不阻塞任务继续
