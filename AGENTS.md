# notify_lark 通知规则

## 核心规则

**所有用户通知统一走飞书。** 不要依赖系统通知——系统通知可能被错过。

## 强制调用时机

下列场景 **必须** 调用 `notify_lark`，不允许跳过：

| 场景 | 格式 |
|---|---|
| 完成任何文件操作（edit/write/rm 等） | `notify_lark "完成: <改了什么>"` |
| 完成多步任务（所有 todo 标记完成） | `notify_lark "任务完成: <摘要>"` |
| 向用户提问或请求选择时 | `notify_lark "需要确认: <问题>"` |
| 执行时间 > 5秒的命令后 | `notify_lark "完成: <操作>"` |
| 遇到需用户处理的错误 | `notify_lark "错误: <描述>"` |

## 调用步骤

1. `notify_lark --help` 检查已安装（通常已全局安装）
2. `notify_lark "<场景>: <内容>"`
3. 消息 200 字以内
4. 调用失败报错但不阻塞任务

## 自检

如果本次回复中你做了工作但未调用 `notify_lark`，你违反了规则。每次回复末尾自查。
